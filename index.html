Project architecture

This portfolio section highlights a structured telemetry workflow with nested components, inspired by clean system-oriented product interfaces and grayscale visual hierarchy.

01

Edge Collection

The ESP32 is responsible for collecting data from sensors or device inputs, preparing payloads, and sending them in a compact form suitable for MQTT transmission.

02

Brokered Messaging

Mosquitto acts as the central message broker, enabling scalable publish/subscribe communication between the embedded layer and backend consumers.

03

Visualization Layer

The backend stores telemetry in a queryable format and delivers the final output to Grafana, where charts and dashboards expose real-time system behavior.
